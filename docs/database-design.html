Database Design

Notism API persistence layer. PostgreSQL via EF Core 8 (Npgsql), configured through AppDbContext (src/Notism.Infrastructure/Persistence/). Domain-Driven Design: aggregate roots own child entities; child collections cascade-delete; cross-aggregate references use Restrict.

Conventions

AspectConvention
Primary keyId — Guid / uuid, on every table
Timestampstimestamp with time zone, UTC
Moneynumeric(18,2) (surcharges numeric(10,2))
EnumsStored as camelCase strings (varchar), not ints
Soft deleteIsDeleted + DeletedAt; unique indexes filtered on "IsDeleted" = false
AuditCreatedAt, UpdatedAt, CreatedBy, UpdatedBy set by app layer
ConfigInline fluent API in OnModelCreating
Extensionpg_trgm enabled for trigram search

Base classes

All entities derive from Entity; aggregate roots derive from AggregateRoot (adds domain-event support). Shared fields:

public abstract class Entity
{
    public Guid Id { get; set; }
    public DateTime CreatedAt { get; set; }
    public DateTime UpdatedAt { get; set; }
    public bool IsDeleted { get; set; }
    public DateTime? DeletedAt { get; set; }
    public Guid? CreatedBy { get; set; }
    public Guid? UpdatedBy { get; set; }
}

Notable design choices

  • Denormalized snapshots — OrderItem and CartItemCustomisation copy labels/prices so historical records survive catalog edits and deletes (why parent FKs are Restrict/SetNull, not Cascade).
  • Soft delete + filtered unique — Category.Name stays reusable after deletion since the unique index filters IsDeleted = false.
  • Trigram search — GIN indexes on LOWER(Name)/LOWER(Description) back case-insensitive substring food search (pg_trgm).
  • Domain events — aggregate roots raise events dispatched via MediatR after SaveChangesAsync.
  • jsonb — BankingCheckout.CartItemIds persists a List<Guid> via ValueConverter with value-comparison.
