vpc.tf 57 lines

Builds a minimal public-only network:

  • aws_vpc.main — DNS hostnames + DNS support enabled
  • aws_internet_gateway.main — outbound/inbound internet
  • aws_subnet.public — single public subnet in first AZ, map_public_ip_on_launch = true
  • aws_route_table.public — default route 0.0.0.0/0 → IGW
  • aws_route_table_association.public — binds RT to subnet
Tradeoff: No private subnet, no NAT gateway. Saves ~$32/month but EC2 lives directly on the public internet. Acceptable for a single API box; not for tiered architectures.

Direction Port Source/Dest Purpose
ingress 22 0.0.0.0/0 SSH

ingress 443 0.0.0.0/0 HTTPS
egress all 0.0.0.0/0 outbound
Risk: SSH open to the world. Should be restricted to your IP, a bastion, or replaced with SSM Session Manager.

ec2.tf 56 lines

  • AMI: latest Amazon Linux 2023 ARM64 (kernel 6.1) via data "aws_ami"
  • Type: t4g.micro (Graviton ARM, free-tier eligible)
  • Storage: 16 GB gp3 root volume
  • IAM: instance profile attached → app uses role credentials, no static keys
  • user_data: bootstrap script — installs docker + compose plugin (aarch64)
#!/bin/bash
set -e
yum update -y
yum install -y docker
systemctl enable docker
systemctl start docker

mkdir -p /usr/local/lib/docker/cli-plugins
curl -SL https://github.com/docker/compose/releases/latest/download/docker-compose-linux-aarch64 \
  -o /usr/local/lib/docker/cli-plugins/docker-compose
chmod +x /usr/local/lib/docker/cli-plugins/docker-compose
Drift trap: user_data only runs on first boot. Editing it later will not re-run unless you set user_data_replace_on_change = true (which destroys + recreates the instance) or manually cloud-init clean + reboot.

s3.tf 219 lines

Three buckets, all BucketOwnerEnforced (no ACLs), all AES256 + bucket key.

Bucket Purpose Public Access Block Policy / Access
private-notism-storage User uploads (auth-gated) open Allow all s3:Get/Put/Delete if aws:Referer matches localhost or CloudFront domain
public-notism-storage Public assets open Public s3:GetObject for everyone
notism-web-prod SPA static hosting locked (all 4 blocks true) Only CloudFront via OAC SourceArn match

CORS configured on the two storage buckets for localhost:4200 + CloudFront origins.

Weak gating: aws:Referer is a client-supplied header — trivially forged with curl -H. Acceptable as a soft "don't deep-link from random sites" filter, NOT as authentication. Move to presigned URLs for real authz.

cloudfront.tf 76 lines global

  • OAC (Origin Access Control) — modern replacement for legacy OAI; signs requests to S3 with SigV4
  • Origin: notism-web-prod bucket regional domain

Notable Design Choices

Choice Why Tradeoff
Single EC2 + Docker Cheap, simple, ~free-tier No HA, manual deploys, single point of failure
Supabase Postgres (not RDS) Free tier, managed backups External dependency, network latency, vendor lock
ARM Graviton (t4g) ~20% cheaper than x86, free tier CI must build arm64 images
ECR + Docker (not Beanstalk/ECS) Direct control, no abstraction tax No auto-scale or auto-replace on health-fail
CloudFront default cert Free, zero config Can't use custom domain without ACM cert in us-east-1
API on :5000 HTTP No TLS termination cost on EC2 Cleartext over public internet — needs reverse proxy or ALB
Public-only VPC (no NAT) Saves ~$32/month NAT gateway EC2 directly exposed; less defense in depth

Gaps Worth Filing

SSH wide open — security group allows 22/0.0.0.0/0. Restrict to known IPs or migrate to SSM Session Manager (no inbound port at all).
No user_data_replace_on_change — editing the bootstrap script in ec2.tf won't actually re-run on the running instance (already hit this; docker had to be installed manually).
API :5000 over plain HTTP — passwords/tokens in cleartext. Add Caddy or nginx + Let's Encrypt locally, or front with an ALB + ACM cert.
No remote state backend — terraform.tfstate is local. Move to S3 + DynamoDB lock so multiple operators / CI can apply safely.
Referer-gated S3 — fine for soft hotlink prevention, but don't treat it as auth. For real per-user access, switch to presigned URLs minted by the API.
No alerting / observability — no CloudWatch alarms, no log shipping. At minimum: CloudWatch agent for system metrics + alarms on CPU / disk.
