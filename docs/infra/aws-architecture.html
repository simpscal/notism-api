1Overview & Default Tags

Notism runs a small, cost-optimized stack: a single ARM EC2 host running the .NET API in Docker Compose, fronted by an Elastic IP; a static SPA frontend served via CloudFront + S3; an event-driven image-resizing pipeline on Lambda; and container images in ECR. The PostgreSQL database is external (Supabase), not AWS. Deploys run from GitHub Actions via OIDC federation (no static AWS keys).

Default tags (provider.tf) — applied to every resource that supports tagging:
  • Project notism
  • Environment prod (var.environment)
  • ManagedBy terraform
Region split
Almost everything lives in ap-northeast-1 (Tokyo). CloudFront and IAM are global. Lambda was intentionally co-located with S3 in ap-northeast-1 to avoid cross-region S3 connection failures.
Architecture
All compute is arm64 / Graviton — EC2 t4g.micro, Lambda arm64, AL2023 ARM AMI, arm64 docker-compose binary.
State note
OIDC provider and deploy roles use Terraform import {} blocks — they pre-existed in the console and were reconciled into state, not created fresh.

Source: vpc.tf, variables.tf. A minimal single-AZ public-only network.

Topology characteristics
  • Single AZ, single subnet, public only. No private subnets, no NAT gateway, no second AZ — minimizes cost, sacrifices HA.
  • EC2 sits directly in the public subnet with a public IP (then an EIP overrides it).
  • No VPC endpoints — S3/ECR access from EC2 goes out via the IGW (public AWS endpoints).

4Compute — EC2 API Host + Elastic IP

Source: ec2.tf, eip.tf. One instance runs the entire .NET API via Docker Compose.

EC2 instance — aws_instance.api (notism-api)

  • Instance type t4g.micro (Graviton2 ARM, burstable, 2 vCPU / 1 GiB)
  • AMI Amazon Linux 2023 ARM64, kernel 6.1 (most_recent data lookup, owner amazon)
  • Subnet public subnet (aws_subnet.public)
  • Security group notism-ec2-sg
  • Instance profile notism-ec2-profile → notism-ec2-role
  • Key pair notism-api (var.key_name)
  • Root volume 16 GiB gp3 EBS, tagged prod/notism/terraform
  • lifecycle ignore_changes = [ami] — new AMI publish won't force replace

user_data bootstrap: yum update, install + enable Docker, then install the docker compose CLI plugin (aarch64 binary) into /usr/local/lib/docker/cli-plugins. The API container itself is pulled from ECR and run by CI/CD, not user_data.

Elastic IP — aws_eip.api + aws_eip_association.api

  • aws_eip.api VPC-domain EIP notism-api-eip — stable public IP
  • aws_eip_association.api binds the EIP to the EC2 instance

Outputs expose ec2_public_ip and api_url = http://<eip>:5000. API listens on port 5000 (note: SG opens 80/443, not 5000 — see security note below).

7Storage — Three S3 Buckets

Source: s3.tf. Three buckets with distinct access models. All use AES256 SSE with bucket keys, and BucketOwnerEnforced ownership (ACLs disabled).

private-notism-storage

Upload landing zone + image-pipeline source.

  • Encryption AES256 + bucket key
  • Public access block ALL OFF (false)
  • CORS PUT/GET/DELETE/HEAD from localhost:4200 + CloudFront domain
  • Policy Allow Get/Put/Delete to * gated by aws:Referer (localhost + CF domain)
  • Notifications ObjectCreated → Lambda (avatar/, food/)

public-notism-storage

Resized-image output, publicly readable.

  • Encryption AES256 + bucket key
  • Public access block ALL OFF (false)
  • CORS PUT/GET/DELETE/HEAD from localhost:4200 only
  • Policy PublicReadGetObject — anyone s3:GetObject

notism-web-prod

SPA static hosting origin, locked to CloudFront.

  • Encryption AES256 + bucket key
  • Public access block ALL ON (true) — fully private
  • CORS none
  • Policy Allow s3:GetObject to CloudFront service principal, scoped by AWS:SourceArn = the distribution ARN (OAC)
Security note: the two storage buckets disable all public-access-block protections and grant Principal: "*". private-notism-storage relies only on an aws:Referer header condition — a weak gate (Referer is client-spoofable). public-notism-storage is genuinely world-readable. Only notism-web-prod is properly locked down (OAC + full PAB).

9Lambda — Image Resizing Pipeline

Source: lambda.tf. One Node.js arm64 function with an execution role and a sharp layer. Co-located in ap-northeast-1 with the S3 buckets.

FunctionTriggerMem / TimeoutKey env
notism-image-resizing S3 avatar/ + food/ ObjectCreated 256 MB / 30 s DESTINATION_BUCKET=public-notism-storage, RESIZE_JOBS (per-prefix variants)
Config
  • Runtime declared nodejs22.x in TF, but actual runtime is nodejs24.x — ignore_changes = [runtime] because provider 5.100.0 schema doesn't yet enumerate nodejs24.
  • Handler index.handler, architecture arm64.
  • Layer: sharp:1 (account-published layer ARN, built from aws_caller_identity).
  • Code deployed by CI/CD — ignore_changes = [filename, source_code_hash, layers, runtime]. Terraform owns only config (env, role, mem, timeout).
  • REGION env passed to Node AWS SDK so S3 clients target ap-northeast-1.

Pipeline shape: a single upload-triggered function. RESIZE_JOBS (JSON env) maps a source prefix to the variants written in one invocation — avatar/ → avatar 200²; food/ → food 400² + food-detail 800². A new size or surface is a config edit, not new infrastructure.

13External Dependencies (Non-AWS)

EXTERNAL PostgreSQL — Supabase

Not provisioned in AWS. The connection_string_hint output documents it: Host=<supabase-host>;Database=postgres;Username=postgres;Port=5432;SSL Mode=Require;Trust Server Certificate=true. The real connection string is injected at deploy via the CONNECTION_STRING GitHub secret. The .NET API (EC2) connects out over the SG's open egress.

CLAUDE.md mentions RDS PostgreSQL as "optional" — no RDS resource exists in this Terraform; the live DB is Supabase.

GitHub Actions — the OIDC identity provider / CI runner (token.actions.githubusercontent.com), repos simpscal/notism-api & simpscal/notism-web.
Angular SPA dev — http://localhost:4200 is whitelisted across all CORS rules and the private-bucket Referer policy.

14End-to-End Data Flows

A. Frontend (SPA) delivery

User → CloudFront (HTTPS, edge cache) → OAC-signed GET → notism-web-prod S3 (private). Missing path / 403 → rewritten to /index.html for client-side routing.

B. API request

Client → Elastic IP (port 80/443 → host → API :5000) → EC2 t4g.micro Docker Compose (.NET API) → Supabase PostgreSQL (5432, TLS) for data; → S3 storage buckets via notism-ec2-role for file read/write.

C. Image upload & resize

Client/API uploads original to private-notism-storage under avatar/ or food/ → S3 ObjectCreated event → image-resizing resizes (sharp) and writes every variant for that prefix to public-notism-storage in one invocation (avatar/ → avatar 200²; food/ → food 400² + food-detail 800²). Public resized images served directly (public-read) or referenced by the SPA.

D. CI/CD deploy

API: GitHub Actions (notism-api) → OIDC AssumeRole → notism-api-deploy-role → build & push image to ECR → ec2:StartInstances (wake host) → host pulls image, compose up → CloudFront invalidate.
Web: GitHub Actions (notism-web) → notism-web-deploy → aws s3 sync to notism-web-prod → CloudFront invalidate.
